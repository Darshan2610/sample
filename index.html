Architecture Overview

Developing an adaptive learning path feature that customizes the sequence of educational content and exercises based on each student's performance and progress. The feature dynamically adjusts to provide more practice on weaker areas and introduces new topics as the student improves.

Database Schema

Students Table:

  • student_id (Primary Key)
  • name
  • email
  • current_level

Content Table:

  • content_id (Primary Key)
  • title
  • description
  • level

Performance Table:

  • performance_id (Primary Key)
  • student_id (Foreign Key)
  • content_id (Foreign Key)
  • score
  • timestamp

Progress Table:

  • progress_id (Primary Key)
  • student_id (Foreign Key)
  • content_id (Foreign Key)
  • status (e.g., 'completed', 'in-progress')

API Endpoints

GET /students/{id}

Fetch student details and current level.

GET /content/next

Fetch next content based on student’s current level and performance.

POST /performance

Record the student’s performance on a piece of content.

GET /progress/{student_id}

Fetch progress details of a student.

POST /progress

Update progress status of a student on a content.

Front-end Components

Dashboard Component:

Display student’s current progress and performance metrics. Provide navigation to the next recommended content.

Content Viewer Component:

Display educational content and exercises. Include interactive elements for submitting responses.

Performance Tracker Component:

Allow students to view their performance history. Provide feedback and additional resources based on performance.

Explanation of Design Choices

Database Schema:

The Students Table stores basic student information and their current learning level. The Content Table contains educational content and the level it is associated with. The Performance Table records each student’s performance, allowing us to track progress and identify weaker areas. The Progress Table tracks the status of content completion for each student, ensuring the system knows what content has been attempted.

API Endpoints:

The API layer provides necessary endpoints to fetch student data, retrieve the next piece of content, record performance, and update progress. These endpoints help in deciding the next content, recording how well a student performed, fetching progress details, and updating the progress status.

Front-end Components:

The Dashboard Component provides an overview of the student's progress and performance, helping them understand their learning journey. The Content Viewer Component displays the educational content and exercises, forming the core interaction point for students. The Performance Tracker Component allows students to see their past performance and receive feedback, encouraging self-assessment and improvement.
